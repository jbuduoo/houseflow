# 住通地圖情報採集系統 - 開發憲法 (v1.0)

為了保護開發成果，防止欄位位移或數據遺失，所有後續開發必須嚴格遵守本憲法。

## 一、 試算表欄位對照表 (A-V 欄位定義)

這是目前最穩定的「金標規格」，未經使用者明確要求，不准更動順序：

| 欄位 | 名稱 | 內容說明 |
| :--- | :--- | :--- |
| **A** | ID | 物件 A10OnLineId |
| **B** | 案件名稱 | 完整標題 |
| **C** | 案件首圖 | 圖片 URL |
| **D** | 物件地址 | **精準地址 (脫殼版，僅保留 新北市...路)** |
| **E** | 售價(萬) | 純數字價格 |
| **F** | 總坪數 | 純數字坪數 |
| **G** | 類型 | **電梯/公寓/大樓/車位/店面 等類型** |
| **H** | 樓層 | 物件所在的當前樓層 |
| **I** | 總樓層 | 該建物的總樓層 |
| **J** | 格局 | X房X廳X衛 |
| **K** | 網頁連結 | **同行原始網址 (591、信義等)** |
| **L** | 比對地址 | **住通系統詳細頁連結** |
| **M** | 查地址 | 詳細門牌 (自動補完) |
| **N** | 查戶籍 | **查閱連結 (單筆時自動填入)** |
| **O** | 戶籍地址 | 戶籍地址 (預留) |
| **P** | 抓取時間 | 執行深挖的時間 |
| **Q-T** | 預留 | (目前為空，供未來擴充) |
| **U** | 委託狀態 | 是否已委託 (Y/N) |
| **V** | 更新時間 | 採集當下的日期時間 |

## 二、 開發三大鐵律 (Safety Protocols)

### 1. 憲法優先 (Mapping Stability)
任何針對 `houseflow_map_scraper.py` 的修改，必須先對照上述表格。嚴禁在未經許可的情況下重新定義 `row_data` 的索引位置。

### 2. 增量修補 (Incremental Patching)
除非是系統性重構，否則每次修改僅針對故障點或新增欄位進行「局部更新」。同步至 Google Sheets 時，應優先使用 `update_cells` 鎖定特定欄位，而非覆蓋整列。

### 3. 數據防護 (Sanity Check)
同步邏輯中必須包含 `if val:` 檢查。**絕對禁止** 用抓取失敗產生的空字串 (`""`) 覆寫試算表中原本已有的正確資料。
