# 工作說明 5_0430: OCR 相關程式整理

## 1. 核心程式碼位置
OCR 相關邏輯目前主要整合在 **`core\4_registry.py`** 中，主要用於處理網頁上以圖片顯示的地址資訊。

- **檔案路徑**: `core\4_registry.py`
- **主要功能**: 當「建築物所有權部」的住址欄位不是文字而是圖片時，自動擷取圖片並進行文字辨識 (OCR)。

## 2. OCR 技術架構
- **辨識引擎**: 使用 `easyocr` (支援繁體中文 `ch_tra` 與英文 `en`)。
- **影像處理**: 使用 `PIL` (Pillow) 進行預處理。
- **依賴套件**: `easyocr`, `pillow`, `numpy`。

## 3. 核心處理邏輯
在 `core\4_registry.py` 中，OCR 流程如下：

### A. 引擎初始化 (`get_ocr_reader`)
```python
# 初始化支援繁體中文與英文的 OCR 引擎
_ocr_reader = easyocr.Reader(['ch_tra', 'en'], gpu=False, verbose=False)
```

### B. 影像預處理 (`ocr_from_img_src`)
為了提升辨識準確率，程式對擷取到的 Base64 圖片進行了以下處理：
1. **放大 4 倍**: 解決原圖解析度過低的問題。
2. **灰階化**: 轉為黑白。
3. **對比度提升**: 增強 3.0 倍。
4. **銳化**: 增強 2.0 倍。

### C. 文字後處理與校正
辨識完成後，會進行常見錯字與格式修正：
- 移除所有空格。
- 校正錯字：例如 `耋` -> `臺`。
- 符號修正：例如 `+` -> `十` (處理「十四樓」誤判)、英文字母誤判為括號等。
