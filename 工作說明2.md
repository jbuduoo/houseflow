# 工作說明 2 - 房仲爬蟲測試模組分析與整合評估

## 一、 新模組功能分析 (`房仲爬蟲測試/house_flow_main.py`)
經過對 `房仲爬蟲測試` 資料夾內的程式進行分析，該模組主要是一個獨立的自動化爬蟲與地址反查工具，包含以下核心功能：

1.  **名單讀取與自動過濾 (`utils.py`)**：
    *   自動讀取根目錄下的 `名單.md` (或 `.txt`)。
    *   透過正規表示式萃取網址，並利用網域分析 (domain extraction) 自動辨識該網址屬於哪一家房仲公司。
2.  **精準座標抓取 (`sinyi.py`)**：
    *   目前專門針對「信義房屋」的網址進行解析，爬取網頁內部隱藏的經緯度座標。
    *   (程式中預留了未來擴充其他房仲的彈性)。
3.  **地址反查與快取機制 (Reverse Geocoding) (`geocoder.py`)**：
    *   將抓取到的經緯度，打 API 反查為實際的物理地址。
    *   內建**快取機制 (Cache)**，若相同的座標已查詢過，會優先從本地快取讀取，避免重複消耗 API 額度並提升速度。
4.  **自動產出整合報表**：
    *   將所有成功抓取與反查的資料，彙整並輸出成 Markdown 格式的 `最終整合報表.md`，方便閱讀與後續處理。

## 二、 整合至 `1_master_run.py` 的討論與方案
要將此新程式整合進 `1_master_run.py`，我們需要決定它在整個「住通自動化地產開發總管」管線中的**定位**。目前舊有管線 (功能 1~5) 是高度依賴 **Google 試算表** 進行資料流轉。而這個新模組目前是**基於本地檔案 (`名單.md` -> `最終整合報表.md`)** 的獨立運作。

這裡提出幾個整合方案供我們討論：

### 方案 A：作為獨立的工具模組 (最快、變動最小)
*   **作法**：在 `1_master_run.py` 新增功能選單 `[6] 🕸️ 房仲名單獨立爬蟲 (本地檔案處理)`。
*   **檔案調整**：可以直接呼叫 `房仲爬蟲測試/house_flow_main.py` 的 `main()`。
*   **優點**：不影響現有的試算表流程，適合用來做臨時批次的名單轉換與座標查詢。
*   **需要解決的問題**：因為 `1_master_run.py` 是在根目錄執行，需要微調 `house_flow_main.py` 內讀取 `名單.md` 與輸出報表的路徑處理 (確保它能在根目錄正確找到檔案，或強制切換工作目錄)。

### 方案 B：核心模組大融合 (架構最乾淨)
*   **作法**：
    1. 將 `房仲爬蟲測試/core/` 下的 `sinyi.py`, `geocoder.py`, `utils.py` 搬移並合併到主程式的 `core/` 資料夾中。
    2. 將 `house_flow_main.py` 改名為 `core/5_spider.py` (或其他適當名稱)。
    3. 在 `1_master_run.py` 加入對應選項。
*   **優點**：所有核心工具統一集中在同一個 `core` 裡，未來其他模組要呼叫反查或抓座標會更方便。

### 方案 C：資料流深度整合 (未來發展方向)
*   **作法**：不僅合入 `1_master_run.py`，更進一步改寫 `house_flow_main.py` 的輸入與輸出。
    *   **輸入**：不再讀取本地的 `名單.md`，而是改去掃描 Google 試算表中特定欄位的網址。
    *   **輸出**：不只產生 `最終整合報表.md`，更直接把爬到的座標與反查地址**寫回** Google 試算表中對應的儲存格。
*   **優點**：實現資料完全自動化的閉環。
