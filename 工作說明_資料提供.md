# 工作說明：資料提供與地址比對邏輯

本文件旨在探討與規劃「住通地圖開放系統」中，兩種不同來源的地址資料該如何解讀，以及未來如何合併為「綜合結論」呈現給終端使用者（房仲業務員）。

## 一、 資料來源與本質差異

一個物件會得到兩個不同維度的地址資訊：

1. **查地址 (產權/戶籍邏輯)：** 利用網路上的「坪數、樓層、總樓高」去比對系統產權資料。這是物件的「真實身分證」，通常精確包含樓層。
2. **反查地址 (地理/空間邏輯)：** 透過網頁打卡點的「經緯度座標」回推實體門牌。這反映了「大頭針插在哪裡」，但房仲常為了防同行而故意插偏（假座標）。

---

## 二、 查址結果與定位命中矩陣 (綜合結論回覆表)

針對「查地址」的四種狀態（百分百符合、疑似一筆、多筆符合、查無）與「定位反查」是否命中（吻合/不吻合），系統將依照以下矩陣直接給出最終結論。

*(名詞定義：「命中」代表定位反查的門牌，與查地址推算出的門牌號碼相符)*

| 產權查址結果 | 定位 (反查) 結果 | 最終系統回覆 / 呈現給使用者的白話文 | 邏輯意義 |
| :--- | :--- | :--- | :--- |
| **百分百符合**<br>*(精確命中一筆)* | **✅ 吻合命中**<br>*(定位就是那間)* | `🎯 雙重確認：【中和路10號5樓】`<br>*(備註：產權與網頁座標完美重合，請安心開發)* | 極度可信。原業務員誠實標示且無隱藏座標。 |
| **百分百符合**<br>*(精確命中一筆)* | **❌ 不吻合(偏移)**<br>*(定位在其他號)* | `🎯 真實門牌：【中和路10號5樓】`<br>*(⚠️ 注意：網頁座標刻意偏移至 20 號，請忽略座標)* | 產權邏輯難以造假，因此**絕對信任查址結果**。判定網頁上為防守用假座標。 |
| **百分百符合**<br>*(精確命中一筆)* | **❓ 無座標** | `🎯 系統推算：【中和路10號5樓】`<br>*(備註：無地圖定位可驗證，請依此門牌尋訪)* | 單方面依賴產權推算，依然具備極高準確度。 |
| **疑似一筆**<br>*(僅一筆但不完全符合)* | **✅ 吻合命中**<br>*(定位剛好吻合)* | `🎯 雙重確認：推薦為【中和路10號5樓】`<br>*(備註：產權比對為疑似，但座標命中補足了信賴度)* | **超級互補！** 雖然產權有點瑕疵（可能坪數微調），但座標打在同一間，兩邊情報疊加讓可信度大幅升級。 |
| **疑似一筆**<br>*(僅一筆但不完全符合)* | **❌ 不吻合(偏移)**<br>*(定位在其他號)* | `❓ 疑似門牌：【中和路10號5樓】`<br>*(⚠️ 注意：產權未100%吻合，且座標定於 16 號，兩者皆需存疑)* | 兩邊的證據都不夠堅固，這時候不能隨便幫業務下結論，請業務自行判斷要看 10 號還是 16 號。 |
| **疑似一筆**<br>*(僅一筆但不完全符合)* | **❓ 無座標** | `❓ 疑似門牌：【中和路10號5樓】`<br>*(備註：產權未100%吻合，且無地圖定位可驗證)* | 線索薄弱，僅供初步參考。 |
| **有多筆符合**<br>*(如: 10,12,14號)* | **✅ 吻合命中**<br>*(定位命中10號)* | `🎯 座標助攻：推薦為【中和路10號】`<br>*(備註：產權符合多筆，但座標精確命中 10 號)* | 用座標解決了產權資料的最後一哩路，幫業務做出最終選擇。 |
| **有多筆符合**<br>*(如: 10,12,14號)* | **❌ 不吻合(偏移)**<br>*(定位在16號)* | `🔍 真實門牌應為：【中和路10, 12, 14號】之一`<br>*(⚠️ 注意：座標定位於 16 號，未命中產權清單，僅供參考)* | 產權說絕對是 10/12/14 之一，座標卻說是 16。此時視為座標微偏，請業務往 10~14 號尋找。 |
| **查無地址**<br>*(查無產權資料)* | **✅ 有反查門牌**<br>*(定位在10號)* | `📍 地圖定位：疑似在【中和路10號】附近`<br>*(備註：產權條件有誤導致查無資料，請至實地尋訪)* | 產權資料失效，只能死馬當活馬醫，去打卡點碰運氣。 |
| **查無地址**<br>*(查無產權資料)* | **❌ 查無反查結果** | `❌ 查無有效開發資訊`<br>*(備註：產權與座標皆無法取得)* | 完全沒有線索。 |

## 三、 給開發者的系統實作建議

為了落實上述的「回覆矩陣」，我們建議在 Python 爬蟲與整合腳本的最後階段，加入一隻 `address_validator.py` 模組：

1. **萃取門牌數字：** 用正則表達式把「查地址」的號碼（例如 `10`, `12`, `14`）與「反查地址」的號碼提取出來。
2. **交集比對 (Intersection)：** 如果反查的數字，存在於查地址的數字清單中，就判定為 **「✅ 吻合命中」**；反之則為 **「❌ 不吻合」**。
3. **自動輸出白話文：** 將這句話直接寫入 Google Sheet 的新欄位（例如：`Z欄：系統綜合研判`），讓業務員打開 App 或試算表時，第一眼就只看這句話。
