# 1_master_run.py — 選單化改造規劃

## 目標
將現有的固定三階段流水線，改為**互動式選單**，讓使用者可以自由選擇要執行哪幾個功能、以何種順序執行。

---

## 🔍 功能盤點 (模組詳細說明)

共有 **5 個核心功能模組**，加上「全部執行」共 6 個選項：

| 編號 | 功能名稱 | 檔案位置 | 說明 |
|------|----------|----------|------|
| **1** | 🔍 採集物件 | `core/1_house.py` | **第一線開發**：開啟 Playwright 瀏覽器，從 HouseFlow 地圖抓取物件基礎資料（價格、坪數、圖片、來源仲介），並存入試算表 A~V 欄。 |
| **2** | 🗑️ 去除重複 ID | `core/utils_deduplicate.py` | **資料品質**：掃描全表，針對重複 ID，**保留來源仲介等級最高的那一筆**（如：信義 > 永慶 > 591）。 |
| **3** | 📍 地址補完 | `core/2_address.py` | **精準定位**：開啟瀏覽器模擬點擊「找房屋」按鈕，從系統內頁擷取真正的門牌號碼（填入 M 欄：查地址）。 |
| **4** | 🏠 戶籍/座標擷取 | `core/3_registry.py` | **深度調查**：開啟瀏覽器點擊「查閱」，透過 OCR 或文字讀取取得**屋主戶籍地址**（填入 O 欄），並呼叫 ArcGIS API 補齊物件與戶籍的**經緯度座標**。 |
| **5** | 🗺️ 補齊座標 (純補) | `core/4_coords.py` | **快速修復**：純後台 API 呼叫。當已經有地址但遺失座標時，不開瀏覽器直接補齊 R、S、T、U 欄位。 |
| **0** | ⚡ 全部依序執行 | - | 懶人模式。依序執行：**1 ➔ 2 ➔ 3 ➔ 4**。 |

---

## 🛠️ 改動實作細節 (技術面)

### 1. 互動式選單設計
執行 `1_master_run.py` 後會停下來詢問：
> 「請輸入功能編號（例如：1 2 4 或 0）：」
- 支援空格多選，並按輸入順序執行。
- 加入 `time.time()` 紀錄各階段耗時。

### 2. 整合機制
- 使用 `importlib` 動態載入 `core/` 下的檔案。
- 修復 `core/4_coords.py` 遺漏 `import os` 的 Bug。

## ❓ 待確認問題 (Open Questions)

> [!IMPORTANT]
> **Q1：功能 5 (純補座標) 與功能 4 的重疊**
> - *決議：保留功能 5。因為功能 5 作為「快速修復」選項，速度快且不需要瀏覽器登入。*

> [!IMPORTANT]
> **Q2：重複資料 (選項 2) 的自動化程度**
> - 在「全部執行 (0)」模式下，是否要跳過 y/n 確認，直接自動執行？
> - *決議：因為有時會有需要輸入授權碼的狀態，所以沒有辦法跳過，維持手動確認。*
