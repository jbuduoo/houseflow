## [2] 去除重複 ID (`core/utils_deduplicate.py`)
### 去重邏輯與順序
程式採用 **「來源可靠性優先」** 的策略進行去重：

1.  **讀取順序**: 依據試算表由上而下的順序掃描。
2.  **ID 比對**: 以第一欄「物件ID」為基準。
3.  **仲介權重**: 當發現重複 ID 時，會比對 **V 欄 (來源仲介)**。
    *   優先序：**信義(1) > 永慶(2) > 好房(3) > 住商(4) > 591(5) ...**
    *   如果新掃描到的資料來源等級「高於」已存資料（數字較小），則會用新資料**取代**舊資料。
4.  **結果**: 最終每個 ID 只會保留一筆「最可靠來源」的資料。

## [4] 戶籍/座標擷取 (`core/3_registry.py`)
### 擷取方式
深入地籍資料頁面進行精準提取：

1.  **定位區塊**: 使用 `TreeWalker` 技術精確定位到 **「建築物所有權部」** 區塊。
2.  **欄位掃描**: 尋找「住址」或「住所」欄位。
3.  **OCR 支援**: 
    *   若住址是文字，直接抓取。
    *   若住址被圖片化（防爬機制），則自動啟動 **EasyOCR** 進行影像辨識。
4.  **地址清理**: 移除「里/鄰/樓層」等干擾資訊，並標準化縣市名稱（如：台北縣 ➔ 新北市）。
5.  **輸出**: 寫入 **O 欄 (戶籍地址)**。
